Obsidian Team Vault
管理员操作手册

零、每次开机

  1. 打开 Obsidian
  2. 打开终端窗口1 — 代理服务
  3. 打开终端窗口2 — 隧道

窗口1：

cd ~/macobisidian && python3 -m uvicorn main:app --host 0.0.0.0 --port 8000

窗口2：

cloudflared tunnel --url http://localhost:8000

窗口2 打印的公网地址记下来通知团队。

两个窗口不能关。地址每次重启会变。

一、获取公网地址

grep "trycloudflare.com" ~/macobisidian/tunnel.log | tail -1

二、用户审批

待审列表

curl https://<地址>/admin/pending \
  -H "Authorization: Bearer admintoken"

审批通过

curl -X POST https://<地址>/admin/approve/<用户id> \
  -H "Authorization: Bearer admintoken" \
  -H "Content-Type: application/json" \
  -d '{"role": "reader"}'

返回 {"token":"xxx"}，发给该用户。

拒绝申请

curl -X POST https://<地址>/admin/reject/<用户id> \
  -H "Authorization: Bearer admintoken"
角色权限
reader只读
editor读写
admin全部 + 管理，Token 可直接读写 Vault

三、用户管理

所有用户

curl https://<地址>/admin/users \
  -H "Authorization: Bearer admintoken"

撤销用户

curl -X DELETE https://<地址>/admin/revoke/<用户id> \
  -H "Authorization: Bearer admintoken"

四、管理员读写 Vault

管理员 Token 可直接操作 Obsidian：

curl https://<地址>/vault/笔记路径.md \
  -H "Authorization: Bearer admintoken"

五、管理员 Token

~/macobisidian/.env 中的 ADMIN_TOKEN，当前为 admintoken。修改后重启窗口1。

六、安全

  • 管理员 Token 不发给普通用户
  • 离职立即撤销
  • 窗口关闭即停服务

开发文档

架构

客户端 ──HTTPS──→ Cloudflare Tunnel ──→ Auth Proxy (:8000) ──→ Obsidian REST API (:27124)
                                            │
                                        users.db

文件清单

文件作用
main.py代理：注册、鉴权、角色控制、转发
.envOBSIDIAN_URL / OBSIDIAN_API_KEY / ADMIN_TOKEN
users.dbSQLite，首次运行自动创建

权限模型

reader  → GET
editor  → GET PUT POST PATCH DELETE
admin   → 全部 + /admin/* + 可直接用 Token 读写 Vault

API

公开

POST /register             提交申请

用户（需 Token）

GET    /                       健康检查
GET    /vault/{path}           读文件
PUT    /vault/{path}           写文件
POST   /vault/{path}           追加
PATCH  /vault/{path}           定向编辑
DELETE /vault/{path}           删除
POST   /search/simple/?query=  搜索

管理员（需 ADMIN_TOKEN）

GET    /admin/pending          待审列表
GET    /admin/users            所有用户
POST   /admin/approve/{id}     审批+生成Token
POST   /admin/reject/{id}      拒绝申请
DELETE /admin/revoke/{id}      撤销用户

users.db 表结构

id             INTEGER PK
name           TEXT
purpose        TEXT
requested_role TEXT
role           TEXT
token          TEXT UNIQUE
status         TEXT       -- pending | approved | rejected | revoked
created_at     TIMESTAMP

鉴权流程

  1. POST /register → status=pending
  2. POST /admin/approve/{id} → 生成 token, status=approved
  3. Proxy: token 匹配 ADMIN_TOKEN → 直接放行；否则查 users.db → 匹配 role → 匹配 method → 转发
  4. DELETE /admin/revoke/{id} → token 置 NULL, status=revoked

扩展

  • 加角色：改 ROLE_METHODS 和 Role enum
  • 加管理员接口：写在 /{path:path} catch-all 之前
  • 加通知：在 register/approve 路由中插入 webhook

依赖

Python 3.9+, fastapi, uvicorn, httpx, python-dotenv, pydantic
